# Google Developer Documentation Style Guide

## Purpose

To provide editorial guidelines for writing clear and consistent technical documentation for software developers and other technical practitioners.

## Audience

Software developers and technical practitioners.

## Tone

Professional and clear, prioritizing clarity and consistency.

## Voice

Direct and instructional, with flexibility to deviate for clarity and consistency.

## Writing Principles

- Prioritize clarity and consistency.
- Use project-specific guidelines first, then this guide, and finally third-party references.
- Break rules if it improves content clarity and consistency.

## Headings

### Heading Rules

- Use headings to organize content logically.

### Capitalization Rules

- Follow standard capitalization rules for headings.

## Terminology

### Preferred Terms

- Use terms that are clear and widely accepted within the technical community.

### Avoided Terms

- Avoid jargon unless necessary and well-explained.

### Terminology Examples

- Use the word list provided in the guide for specific terminology guidance.

## Grammar

### Active / Passive Voice

- Prefer active voice for clarity.

### Pronouns

- Use second person pronouns for direct address.

### Sentence Length

- Keep sentences concise to enhance readability.

## Lists

### Bulleted Lists

- Use for unordered items.

### Numbered Lists

- Use for sequential steps or ordered items.

## Procedures

### Task Writing

- Clearly outline tasks with necessary context.

### Step Writing

- Use numbered steps for clarity in procedures.

## User Interface Language

### Buttons

- Use clear and descriptive labels.

### Menus

- Ensure menu items are intuitive and descriptive.

### Dialogs

- Write dialogs that are clear and provide necessary information.

### Messages

- Ensure messages are concise and informative.

## Accessibility

### Inclusive Language

- Use language that is inclusive and respectful.

### Bias-Free Language

- Avoid language that could be biased or discriminatory.

### Alt Text

- Provide descriptive alt text for images to ensure accessibility.

## Formatting

### Notes

- Use notes to highlight important information.

### Warnings

- Clearly format warnings to stand out and convey urgency.

### Code Examples

- Ensure code examples are clear and relevant.

### Links

- Use descriptive link text for clarity and accessibility.

## Examples

- Provide examples to clarify complex concepts.

## Exceptions

- Deviate from guidelines when necessary for clarity and consistency, ensuring consistency throughout the document.
