# Microsoft Writing Style Guide

## Purpose
The Microsoft Writing Style Guide provides guidance on writing style and terminology for all forms of communication related to computer technology.

## Audience
The guide is intended for anyone writing about technology, regardless of their role.

## Tone
The tone is warm, relaxed, crisp, and clear, adapting from serious to empathetic to lighthearted based on context.

## Voice
Microsoft's voice is simple and human, focusing on personality, substance, tone, and style. It emphasizes being warm, relaxed, crisp, clear, and ready to assist.

## Writing Principles
- Get to the point quickly.
- Use conversational language.
- Avoid jargon and acronyms.
- Use short sentences and fragments for clarity.
- Use contractions for a friendly tone.

## Headings

### Heading Rules
- Use sentence-style capitalization.
- Avoid end punctuation for titles, headings, and subheadings.

### Capitalization Rules
- Capitalize only the first word and proper nouns in headings.
- Use title-style capitalization for product names and titles in citations.

## Terminology

### Preferred Terms
- Use "allow list" and "block list" instead of "whitelist" and "blacklist."
- Use "choose" instead of "select" when UI text begins with "Select."

### Avoided Terms
- Avoid "master/slave" terminology.
- Avoid "press and hold" and "right-click" in favor of input-neutral verbs.

### Terminology Examples
- Use "allow list" instead of "whitelist."
- Use "block list" instead of "blacklist."

## Grammar

### Active / Passive Voice
- Use active voice whenever possible.

### Pronouns
- Use second person (you) and avoid gender-specific pronouns.

### Sentence Length
- Use short sentences for clarity and ease of reading.

## Lists

### Bulleted Lists
- Use for items that do not need to be in a specific order.

### Numbered Lists
- Use for step-by-step instructions or when order is important.

## Procedures

### Task Writing
- Use numbered steps for procedures.
- Write complete sentences for each step.

### Step Writing
- Use imperative verbs.
- Include actions that complete a procedure.

## User Interface Language

### Buttons
- Use verbs like "Select" and "Enter" for button actions.

### Menus
- Use "Go to" for navigating menus.

### Dialogs
- Use "Open" and "Close" for dialogs.

### Messages
- Use clear and concise language for messages.

## Accessibility

### Inclusive Language
- Avoid terms like "special needs."
- Use people-first language where appropriate.

### Bias-Free Language
- Avoid gender-specific language and use neutral terms.

### Alt Text
- Provide clear and descriptive alt text for images.

## Formatting

### Notes
- Use for additional information that is not critical.

### Warnings
- Use for important information that prevents errors.

### Code Examples
- Use clear and concise examples with proper formatting.

### Links
- Use descriptive link text and avoid "click here."

## Examples
- Use real-world examples to illustrate points.

## Exceptions
- Use title-style capitalization for product names and titles in citations.
- Use an en dash for ranges of pages or where space is limited.
